# Концепция проекта: Telegram-бот «Селена» — персональный астролог

## Проблема и решение

### Проблема
Женщины 25–45 лет в точках жизненного перелома ищут понимания, подтверждения и направления. Традиционная астрология либо слишком академична, либо слишком поверхностна. Качественный персональный астролог — дорого и недоступно. Существующие боты и приложения дают обезличенные гороскопы без учёта реальных данных рождения.

### Решение
Telegram-бот «Селена» — персональный астролог с характером. Бот:
- Собирает реальные данные рождения через удобный wizard-интерфейс
- Генерирует персонализированный натальный гороскоп на основе этих данных через LLM
- Запоминает пользователя — не нужно вводить данные снова
- Общается тёплым, живым русским языком через продуманный персонаж

---

## Целевая аудитория

Русскоязычные женщины 25–45 лет, переживающие точки жизненного перелома:

| Сегмент | Возраст | Боль |
|---------|---------|------|
| «Ищущие» | 25–29 | «Кто я?», давление «часиков», работа не по призванию |
| «Тревожные» | 28–33 | Кризис 30 лет, синдром самозванца, «правильно ли я делаю?» |
| «Выгоревшие» | 30–38 | День сурка, выдохшиеся отношения, «когда закончится эта полоса?» |
| «Перестройщицы» | 35–45 | Переоценка брака/карьеры/идентичности, «ещё не поздно?» |

**Что они хотят услышать:**
1. Подтверждение: «ты на правильном пути»
2. Объяснение страдания через астрологическую рамку
3. Таймлайн: «через [период] начнётся благоприятное время»
4. Разрешение: «твоя карта показывает, что тебе можно рискнуть»
5. Уникальность: «у тебя редкая комбинация»
6. Предназначение: «ты рождена для конкретного дела»

## Ключевые фичи MVP

### В скоупе:
- **Wizard-онбординг** через Aiogram 3 FSM: имя → дата рождения → время рождения → место рождения
- **Распознавание вернувшегося пользователя** по telegram_id (не просить вводить данные повторно)
- **Генерация персонального гороскопа** через OpenRouter API с системным промптом Селены
- **Команды бота:**
  - `/start` — приветствие и запуск онбординга (или приветствие вернувшегося пользователя)
  - `/profile` — просмотр сохранённых данных рождения
  - `/horoscope` — запросить персональный гороскоп
  - `/reset` — сбросить профиль и начать заново
- **Хранение профилей** в PostgreSQL: telegram_id, name, birth_date, birth_time, birth_place, created_at
- **Персонаж Селены** — мудрая, тёплая, говорит красивым русским языком, строго следует структуре БОЛЬ → ПОНИМАНИЕ → ОБЪЯСНЕНИЕ → НАДЕЖДА → ДЕЙСТВИЕ

### Вне скоупа (не в MVP):
- Платёжная система и монетизация
- Ежедневные автоматические рассылки
- Расчёт точных астрологических позиций (используем LLM для интерпретации)
- Веб-интерфейс или мини-приложение Telegram
- Мультиязычность
- Администраторская панель
- Интеграция с реальными астрологическими библиотеками (Swiss Ephemeris и т.п.)
- Push-уведомления о транзитах
- История запросов пользователя

## Технический стек и обоснование

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Bot Framework | Aiogram 3 (Python 3.11+) | Лучший async Telegram framework, встроенный FSM, активная поддержка |
| База данных | PostgreSQL + asyncpg | Надёжная реляционная БД, asyncpg — самый быстрый async драйвер для PostgreSQL |
| ORM | SQLAlchemy 2.x (async) | Стандарт индустрии, поддерживает async, удобные миграции через Alembic |
| AI API | OpenRouter API (aiohttp/httpx) | Агрегатор LLM-моделей, гибкая модель, совместим с OpenAI API формату |
| Конфигурация | pydantic-settings + .env | Типизированная конфигурация, валидация переменных окружения |
| Миграции | Alembic | Стандарт для SQLAlchemy, version control для схемы БД |
| Деплой | Railway.com | Простой деплой Python-приложений, встроенный PostgreSQL, автодеплой из Git |
| Логирование | Python stdlib logging | Без лишних зависимостей, достаточно для MVP |

## Метрики успеха MVP

### Технические метрики:
- Бот успешно запускается и отвечает на `/start` за < 2 секунды
- Онбординг проходит без ошибок от начала до конца
- Гороскоп генерируется за < 30 секунд
- Вернувшийся пользователь распознаётся корректно (не проходит онбординг повторно)
- Профиль сохраняется в БД и восстанавливается при `/profile`
- `/reset` корректно удаляет профиль и позволяет пройти онбординг заново

### Продуктовые метрики (для проверки после запуска):
- Completion rate онбординга > 70% (пользователи, начавшие → завершившие)
- Пользователи, запросившие гороскоп после онбординга > 80%
- Вернувшиеся пользователи (через 24+ часов) > 20%
- Положительная обратная связь на гороскоп (субъективно, через реакции)
